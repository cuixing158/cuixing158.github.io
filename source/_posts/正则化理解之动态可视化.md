> 关于正则化的文章是铺天盖地，其中绝大部分都在解释为什么L1或者L2正则化有利于产生稀疏解，防止过拟合？这个问题被讨论论证了N多次，大篇幅的文字叙述可以让读者了解到其背后的原理，但本文不走大众之路，从力求以动态图来直观解释正则化现象，从而让读者更深刻理解其正则的意义！

抛开理论推导验证环节，需要从文字上面理解的强烈推荐参考博文后面的链接文章，本文以实际例子来进行阐述，考虑L1正则化对二次损失函数权值的影响。
假设我们的损失函数：
   <a href="https://www.codecogs.com/eqnedit.php?latex=L&space;=&space;\left&space;(x_1-1)^2&plus;(x_2-2)^2\right" target="_blank"><img src="https://latex.codecogs.com/gif.latex?L&space;=&space;\left&space;(x_1-1)^2&plus;(x_2-2)^2\right" title="L = \left (x_1-1)^2+(x_2-2)^2\right" /></a>
带L1正则化的损失函数为：
    <a href="https://www.codecogs.com/eqnedit.php?latex=J&space;=&space;\left&space;(x_1-1)^2&plus;(x_2-2)^2&plus;\lambda\left&space;\|&space;x&space;\right&space;\|&space;\right" target="_blank"><img src="https://latex.codecogs.com/gif.latex?J&space;=&space;\left&space;(x_1-1)^2&plus;(x_2-2)^2&plus;\lambda\left&space;\|&space;x&space;\right&space;\|&space;\right" title="J = \left (x_1-1)^2+(x_2-2)^2+\lambda\left \| x \right \| \right" /></a>   
  其中 lambda为正则化因子，其值大小影响着x_1,x_2的变化，值越大，x_1,x_2就越趋向于0，从而产生稀疏解，典型的具有特征选择的作用。下面代码逐渐变化lambda的大小，分别画出L和J的损失函数和等值线图，红色点为对应lambda取特定值时候J的最小值。
```matlab
clear;clc;close all;
% 参考：https://blog.csdn.net/jinping_shi/article/details/52433975
%%
f = @(x,y)(x-1).^2+(y-2).^2;
xyinterval = [-5,5];
history_p1 = [];history_p2 = [];% 历史记录最小点坐标

%%
for lambda = 0:10 % 动态调整lambda的值，观察最小点的变动
    h1 = figure(1);grid on;hold on;
    FS1 = fsurf(f,xyinterval,'ShowContours','off','LineStyle','-');
    FS1.FaceAlpha = .8;
    ff = @(x,y)(x-1).^2+(y-2).^2+lambda*abs(x)+lambda*abs(y);
    FS2 = fsurf(ff,xyinterval,'ShowContours','on','LineStyle','none');% 绘图，正则化曲面
    
    % 计算正则化后曲面最小值点
    [minx,fval] = fminsearch(@(x)myfun(x,lambda),[0,0]);
    scatter3(minx(1),minx(2),fval,'filled','MarkerFaceColor','r');
    str = sprintf('最小值点(%.1f,%.1f,%.1f)',minx(1),minx(2),fval);
    text(minx(1),minx(2),fval+15,str,'Color','r');
    history_p1 = [history_p1;minx(1),minx(2),fval];
    plot3(history_p1(:,1),history_p1(:,2),history_p1(:,3),'ro-');
    fprintf('lambda:%d,最小值点(%.1f,%.1f,%.1f)\n',lambda,minx(1),minx(2),fval);
    
    legend({'实线为正则化前代价函数','虚线为正则化后代价函数','当前最小点'},'Location','northwest','AutoUpdate','off');
    axis([xyinterval,xyinterval]);view(3);
    
    % 画二维等高线
    h2 = figure(2);grid on;hold on;
    fc1 = fcontour(f,xyinterval,'-','LineWidth',2);% 实线正则化前
    fc2 = fcontour(ff,xyinterval,'--','LineWidth',2); % 虚线正则化后
    scatter(minx(1),minx(2),'filled','MarkerFaceColor','r');
    str = sprintf('最小值点(%.1f,%.1f)',minx(1),minx(2));
    history_p2 = [history_p2;minx(1),minx(2)];
    text(minx(1),minx(2),str,'Color','r');
    plot(history_p2(:,1),history_p2(:,2),'ro-');colorbar;
    legend({'实线为正则化前','虚线为正则化后','当前最小点'})
    
    pause(0.5);
    if lambda~=10
        clf(h1);clf(h2);
        legend('off')
    end
end
```

可以看出，当lambda=0时候，即没有进行正则化，和原损失函数一致，在点(1,2)处取得最小值0，随着lambda逐渐增大，lambda=4，x1,x2就被稀疏为0了，代价函数（损失函数）曲面会越来越尖，逐渐偏离原来(1,2)点。
